Markdown 语法初探 (一)

发表于 2017-11-18 | 阅读次数:

博客搭建好了，接下来就要开心的写博客了。第一次 hexo new “postname” 以后生成的 .md 文件就是我们刚 new 的博客文章啦，md 即是markdown的缩写，对其进行编辑，即可编辑我们的文章啦。我选择用sublime text 3 来编辑。

要想开(gao)心(bi)地(ge)写博客，就要首先来熟悉一下Markdown的基本语法。废话不多说，下面总结一下今天所学。

区块元素

标题

一篇可读性较强的文章一般需要一定的层次感，打开一个人的博客，如果层次分明，各级标题一览无余，那么可以很好的帮助读者节省时间，迅速了解到这篇文章是否自己所需。设定标题的级别可以很好的助力这一点。Hexo可以自动在侧栏形成标题栏。赞！

Markdown支持两种标题的语法，类 Setext 和类 atx 的格式，我觉得类 atx 的语法简直不要太浅显容易，果断选择后者。一言以蔽之，就是：

如果你想要此标题为 n 级标题，那就行首写 n 个 #，分别对应1阶～6阶。

比如：

# 这是 H1
## 这是 H2
### 这是 H3
...
###### 这是 H6
代码块

作为程序猿(gou)，博客里面最不缺的就是代码了，那么如果让代码以其原有的形式展现出来，而不是以普通文本的格式展现呢？

很简单：行首缩进 4 个空格或者 1 个制表符。

比如：

This is a code block.
I love coding.
Coding makes me happy.
列表

有层次感的博客需要列表。比如，你想论证 PHP 就是当今世界最好的语言，你可能 blabla 说 PHP 有哪些好处，一二三列出它的好处。

Markdown 支持有序列表 & 无序列表。

有序列表使用 * 或 + 或 — 来标记其后跟的是一个列表。

* Caroline
* Sara
* Venture

等同于：

+ Caroline
+ Sara
+ Venture

等同于：

— Caroline
— Sara
— Venture

有序列表在行首用 数字 + ‘.’ + ‘空格’ 这样的格式。

1. 98
2. 89
3. 76

需要注意的是，行首的数字跟页面实际显示出来的列表序号是不对应的，你在各个列表前面标任意数字，文章实际显示出来的都是1.xx 2.xx 3.xx.

比如，上面的效果等同于下面这个：

4. 98
8. 89
2. 76
区块引用

写文章的时候有需要引用某段文字时，如何显式地表示这是个引用呢？

很简单，在行首写上 > 这个字符，支持嵌套，n 级引用就写几个 >

格式：

> 爱因斯坦说过，天才来自百分之九十九的汗水加百分之一的灵感

效果就是下面这样

爱因斯坦说过，天才来自百分之九十九的汗水加百分之一的灵感

段落分行

嗯…这个很浅显，我暂时也研究地不深，暂时按照平时写文的习惯来就OK了，enter 以后就分段。

写这篇“文章”用时 1h。更具体的请参照http://wowubuntu.com/markdown/index.html
