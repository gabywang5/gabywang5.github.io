短小精悍的shell

发表于 2018-02-23 | | 阅读次数:

为了看懂部分代码，临时学习了一下shell。如题所言，shell 短小精悍，专心一点的话，两个小时就学完了几本语法，能看懂 shell 文件了。下面就 shell 的一些关键语法也是常用语法进行总结。

shell文件命名

shell 文件可以取任何后缀名，可以是 .sh，也可以是 .ah，只要你想都可以。但是本着见后缀如见内容类型的原则，shell 脚本文件最好以sh为后缀名。

shell文件执行

shell 文件执行前需要给文件加执行权限。
chmod +x ./xxx.sh
执行 sh 文件的格式为：
./xxx.sh
注意，./ 是为了告诉系统，就在当前目录下找 xxx.sh 这个文件。如果不加 ./，那 linux 系统会去 PATH 里面找有没有叫 xxx.sh 的，结果就是找不到！出错！

shell变量

定义变量

shell 让我觉得奇怪的地方是，等号 = 的使用不符合标准的程序员有的习惯。
定义变量的时候，直接变量名 website=”google.com”
注意！变量名 website 跟 = 之间不能有空格！而且 = 跟变量值之间也不能有空格！

使用变量

使用一个定义过的变量，在变量名之前加美元符号 $ 就可以了
例如要在屏幕上显示出 website 的值：
echo $website
或者
echo ${website}
加花括号是为了更加容易识别出变量名的边界，推荐一开始就养成加 {} 的习惯！

传递参数

shell 传递参数时直接在命令后面加参数就可以了，比如
./test.sh 1 2 3
参数之间间隔一个空格

文件包含

如果在一个文件中要使用到另外一个文件的话，要怎么包含这个文件呢？
很简单，在包含文件里面加上
source filename
filename 是被包含的文件
或者
. filename
注意！. 跟filename之间有空格！
